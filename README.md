Helidon Vovk Platform

Навчально-тестовий мікросервісний проєкт, створений для перевірки та демонстрації можливостей бібліотеки vovk-data-core та суміжних компонентів екосистеми Vovk. Основною метою проєкту є експериментальна перевірка підходу до уніфікованих запитів до різнорідних джерел даних.

Призначення проєкту

Проєкт розроблено як тестове середовище для оцінки роботи бібліотеки vovk-data у контексті:

- уніфікованого доступу до даних через спільний API;
- роботи з ORM-рівнем абстракції;
- інтеграції файлових, in-memory та реляційних джерел даних;
- реалізації мікросервісної архітектури на базі єдиного підходу до доступу до даних.

Архітектура модулів

Проєкт складається з таких компонентів:

- auth-service — сервіс автентифікації (порт 9100)
- equipment-service — сервіс управління обладнанням (порт 9200)
- inspection-service — сервіс інспекцій (порт 9300)
- common — спільні утиліти та конфігураційні компоненти
- gis — модуль геоінформаційної обробки даних

Технологічний стек

- Helidon 4.3.4
- Java 21
- vovk-data-core 1.0.0-BETA
- vovk-orm 1.0.0-BETA
- PostgreSQL 17
- Kong API Gateway
- Docker / Docker Compose
- JTS (Java Topology Suite)

Запуск проєкту

Для запуску всіх сервісів у контейнеризованому середовищі використовується Docker Compose:

```bash
docker-compose up
```

Примітка

Проєкт має виключно тестовий та дослідницький характер і був створений з метою перевірки концепції уніфікованих запитів до різних типів джерел даних у межах бібліотеки vovk-data.
